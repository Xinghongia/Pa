## 踩坑记录

### 坑1: BeautifulSoup解析器差异

**问题描述**: 同一个HTML，不同解析器结果不同

```python
soup1 = BeautifulSoup(html, 'html.parser')    # 内置解析器
soup2 = BeautifulSoup(html, 'lxml')           # 需要安装lxml
soup3 = BeautifulSoup(html, 'html5lib')       # 需要安装html5lib
```

**原因**: 不同解析器对不规范HTML的处理策略不同

**解决方案**:
1. 统一使用`lxml`解析器（最严格最快速）
2. 使用前验证解析结果是否正确
3. 处理解析异常，切换解析器

```python
def parse_html(html):
    for parser in ['lxml', 'html.parser', 'html5lib']:
        try:
            soup = BeautifulSoup(html, parser)
            return soup
        except Exception:
            continue
    raise Exception("所有解析器都失败了")
```

### 坑2: XPath返回空列表

**问题描述**: XPath表达式正确但返回空列表

```python
tree = etree.HTML(html)
result = tree.xpath('//div[@class="news"]/a/@href')  # 返回[]
```

**原因**: `lxml`自动补全不完整的HTML标签，导致DOM结构变化

**解决方案**:
1. 使用`etree.tostring(tree)`查看实际解析结果
2. 检查路径是否匹配实际DOM结构
3. 尝试更宽松的匹配条件

```python
tree = etree.HTML(html)

# 调试：查看解析后的HTML
# print(etree.tostring(tree, pretty_print=True).decode())

result = tree.xpath('//div[contains(@class, "news")]//a/@href')
```

### 坑3: 正则贪婪匹配过度

**问题描述**: `.*`匹配了过多内容

```python
html = '<div>内容1</div><div>内容2</div>'
re.search(r'<div>.*</div>', html).group()
# 结果: <div>内容1</div><div>内容2</div>  ❌

re.search(r'<div>.*?</div>', html).group()
# 结果: <div>内容1</div>  ✅
```

**解决方案**: 使用非贪婪匹配`.*?`，或精确匹配内容

```python
re.search(r'<div>([^<]*)</div>', html).group(1)
# 结果: 内容1
```

### 坑4: 中文编码问题

**问题描述**: 提取的中文是乱码或`\xa0`等特殊字符

```python
text = soup.select_one('.title').text
# 结果: 'Python \xa0 3.12'  ❌
```

**原因**: HTML中的`&nbsp;`（`\xa0`）等特殊字符

**解决方案**:

```python
import re
import unicodedata

def clean_text(text):
    # 替换\xa0为空格
    text = text.replace('\xa0', ' ')
    # 替换多个空格为单个
    text = re.sub(r'\s+', ' ', text)
    # 去除首尾空白
    return text.strip()

text = clean_text(soup.select_one('.title').text)
```

### 坑5: JSON数据KeyError

**问题描述**: 访问嵌套JSON时KeyError

```python
data = {"user": None}
name = data['user']['name']  # TypeError: 'NoneType' object is not subscriptable
```

**解决方案**:

```python
# 方式1: get()链式调用
name = data.get('user', {}).get('name', '未知')

# 方式2: jsonpath
from jsonpath import jsonpath
result = jsonpath(data, '$.user.name')
name = result[0] if result else '未知'

# 方式3: try-except
try:
    name = data['user']['name']
except (KeyError, TypeError):
    name = '未知'
```

### 坑6: XPath轴函数不支持

**问题描述**: 使用`preceding-sibling::`报错

```python
tree.xpath('//div/preceding-sibling::span')  # ❌ 可能报错
```

**原因**: `preceding-sibling`轴在某些lxml版本中行为不一致

**解决方案**: 使用替代方案

```python
# 替代方案1: 找父元素后遍历兄弟
parent = tree.xpath('//div')[0]
preceding = parent.xpath('span[@class="before"]')

# 替代方案2: 使用position判断
items = tree.xpath('//li')
for i, item in enumerate(items):
    if i > 0:
        previous = items[i-1]
```

### 坑7: CSS选择器:class命名冲突

**问题描述**: 多个元素有相同的class，难以区分

```html
<div class="item">
    <div class="content">
        <span class="name">作者</span>
        <span class="value">张三</span>
    </div>
    <div class="content">
        <span class="name">日期</span>
        <span class="value">2024-01-15</span>
    </div>
</div>
```

**解决方案**:

```python
soup = BeautifulSoup(html, 'html.parser')
items = soup.select('.item')

for item in items:
    # 使用索引选择特定项
    contents = item.select('.content')
    if len(contents) >= 2:
        author = contents[0].select_one('.value').text
        date = contents[1].select_one('.value').text
```

## 常见问题FAQ

### Q1: BeautifulSoup和lxml哪个更快？

**答**: lxml更快，特别是处理大型HTML文档。

| 解析器 | 速度 | 容错性 | 依赖 |
|--------|------|--------|------|
| lxml | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 需要安装lxml |
| html.parser | ⭐⭐⭐ | ⭐⭐⭐ | 内置 |
| html5lib | ⭐⭐ | ⭐⭐⭐⭐⭐ | 需要安装html5lib |

**建议**: 优先使用lxml，遇到解析问题时切换html5lib

---

### Q2: 什么时候用CSS选择器，什么时候用XPath？

**答**: 根据场景选择

| 场景 | 推荐 |
|------|------|
| 简单元素选择 | CSS选择器 |
| 需要父元素 | XPath |
| 需要文本匹配 | XPath |
| 需要条件判断 | 两者皆可 |
| 大量数据处理 | XPath或正则 |

---

### Q3: 正则表达式能完全替代选择器吗？

**答**: 不能完全替代，但可以互补

- 正则：适合文本提取、数据清洗
- 选择器：适合结构化HTML解析

**最佳实践**: 选择器解析HTML结构 + 正则提取文本

---

### Q4: 如何处理动态加载的页面？

**答**: 两种方案

1. **找API接口**（推荐）：
   - 使用Chrome DevTools Network面板
   - 找到XHR/Fetch请求
   - 直接请求API获取数据

2. **Selenium/Playwright**（备选）：
   - 模拟浏览器执行
   - 等待页面渲染完成
   - 获取渲染后的HTML

### Q5: 如何避免被反爬？

**答**: 基础反爬措施

1. 设置合理的User-Agent
2. 添加Referer头
3. 控制请求频率
4. 使用代理IP
5. 处理Cookie/Session
6. 识别验证码并处理

## 性能优化建议

### 1. 解析优化

```python
# ❌ 低效：每次都重新解析
for url in urls:
    html = fetch(url)
    soup = BeautifulSoup(html, 'lxml')
    title = soup.select_one('.title').text

# ✅ 高效：复用解析器
soup = BeautifulSoup('', 'lxml')
for url in urls:
    html = fetch(url)
    soup.clear()
    soup.feed(html)
    title = soup.select_one('.title').text
```

### 2. 选择器优化

```python
# ❌ 低效：重复选择
for item in soup.select('.item'):
    title = item.select_one('.title').text
    date = item.select_one('.date').text
    author = item.select_one('.author').text

# ✅ 高效：使用属性直接提取
for item in soup.select('.item'):
    title = item.get('data-title', '')
    date = item.get('data-date', '')
    author = item.get('data-author', '')
```

### 3. 正则编译

```python
# ❌ 低效：每次都编译
for text in texts:
    re.findall(r'\d{4}-\d{2}-\d{2}', text)

# ✅ 高效：预编译正则
pattern = re.compile(r'\d{4}-\d{2}-\d{2}')
for text in texts:
    pattern.findall(text)
```
