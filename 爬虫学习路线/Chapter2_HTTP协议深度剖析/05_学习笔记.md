## 重点记录

### TCP三次握手

```
握手过程：

客户端                              服务器
  │                                   │
  │ ───── SYN=1, seq=x ───────────→ │
  │                                   │
  │ ←── SYN=1, ACK=1, seq=y, ack=x+1 │
  │                                   │
  │ ───── ACK=1, seq=x+1, ack=y+1 → │
  │                                   │
  连接建立完成                         │

为什么要3次？
- 第1次：客户端确认自己能发
- 第2次：服务器确认自己能收也能发
- 第3次：客户端确认服务器能收
```

### HTTP请求结构

```
请求 = 请求行 + 请求头 + 请求体

请求行：GET /path HTTP/1.1
请求头：Host: xxx / User-Agent: xxx / Cookie: xxx
请求体：（POST请求时才有）
```

### HTTPS加密原理

```
混合加密：
1. 非对称加密（握手）：用公钥加密会话密钥
2. 对称加密（传输）：用会话密钥加密数据
```

### Cookie vs Session

| | Cookie | Session |
|---|---|---|
| 存储位置 | 客户端（浏览器） | 服务器端 |
| 安全性 | 较低 | 较高 |
| 存储量 | 小（<4KB） | 大 |
