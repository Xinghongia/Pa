## 01_http_methods.py - HTTP请求与响应结构

### 教学目标
理解HTTP请求的完整结构（请求行/请求头/请求体）和HTTP响应的完整结构（状态行/响应头/响应体）。

### 核心知识点

```
HTTP请求 = 请求行 + 请求头 + 请求体
HTTP响应 = 状态行 + 响应头 + 响应体
```

### 关键代码解析

**1. 查看完整的请求头**

```python
response = requests.get("https://httpbin.org/get")

print(response.request.headers)
# 输出：
# {'User-Agent': '...', 'Accept': '*/*', 'Accept-Encoding': 'gzip, deflate', ...}
```

**2. POST请求带JSON数据**

```python
data = {"username": "test", "password": "123456"}
response = requests.post(url, json=data)

# 等价于手动设置Content-Type
response = requests.post(url, data=json.dumps(data), headers={"Content-Type": "application/json"})
```

**3. 常见状态码判断**

```python
if response.status_code == 200:
    print("请求成功")
elif response.status_code == 404:
    print("资源不存在")
elif response.status_code == 429:
    print("被限流了，增加间隔")
```

### 运行方式

```bash
python code/01_http_methods.py
```

## 02_cookie_session.py - Cookie与Session管理

### 教学目标
理解Cookie的工作原理，掌握Session自动管理Cookie的方法。

### 核心知识点

```
Cookie = 服务器发给客户端的"通行证"（存在浏览器中）
Session = 服务器存储的用户信息（通过Cookie中的SessionID关联）
```

### 关键代码解析

**1. Session自动保持Cookie**

```python
session = requests.Session()

# 第一次请求设置Cookie
session.get("https://example.com/cookies/set/session/abc123")

# 后续请求自动带上Cookie
response = session.get("https://example.com/user")
# 不需要手动处理Cookie，Session自动帮你管理
```

**2. Cookie持久化**

```python
# 保存Cookie到文件
session.cookies.save("cookies.txt")

# 从文件加载Cookie
session.cookies.load("cookies.txt")
```

**3. 自定义Cookie**

```python
cookies = {"user": "test_user", "token": "abc123"}
response = requests.get(url, cookies=cookies)
```

### 运行方式

```bash
python code/02_cookie_session.py
```

## 03_proxy_auth.py - 代理与认证机制

### 教学目标
理解代理的工作原理，掌握Basic认证、Bearer Token和JWT的原理。

### 核心知识点

```
代理 = 转发请求，隐藏真实IP
Basic认证 = 用户名:密码 → Base64编码
Bearer Token = OAuth2.0中的令牌
JWT = 可验证的JSON Web Token（Header.Payload.Signature）
```

### 关键代码解析

**1. 使用代理**

```python
proxies = {
    'http': 'http://127.0.0.1:7890',
    'https': 'http://127.0.0.1:7890',
}
response = requests.get(url, proxies=proxies)
```

**2. 带认证的代理**

```python
proxy_url = "http://username:password@127.0.0.1:7890"
proxies = {'http': proxy_url, 'https': proxy_url}
```

**3. JWT解码**

```python
import base64
import json

token = "header.payload.signature"
parts = token.split('.')

# 解码Header
header = json.loads(base64.b64decode(parts[0] + '=='))

# 解码Payload
payload = json.loads(base64.b64decode(parts[1] + '=='))
```

### 运行方式

```bash
python code/03_proxy_auth.py
```

## 04_websocket_https.py - WebSocket与HTTPS

### 教学目标
理解HTTPS的加密原理，理解WebSocket与HTTP的区别。

### 核心知识点

```
HTTPS = TLS加密的对称加密通信
       握手用非对称加密 → 传输用对称加密

WebSocket = 双向长连接（服务器可以主动推送）
HTTP = 请求-响应模式（只能客户端轮询）
```

### 关键代码解析

**1. HTTPS请求（默认验证证书）**

```python
response = requests.get("https://example.com")
# 默认验证SSL证书
```

**2. 忽略证书验证（测试用）**

```python
response = requests.get("https://example.com", verify=False)
# 不要在生产环境使用！
```

**3. WebSocket基本用法**

```python
import websocket

ws = websocket.WebSocketApp(
    "wss://example.com/ws",
    on_message=lambda ws, msg: print(f"收到: {msg}"),
    on_error=lambda ws, err: print(f"错误: {err}"),
)
ws.run_forever()
```

### 运行方式

```bash
python code/04_websocket_https.py
```

## 05_connection_pool.py - TCP连接与连接池

### 教学目标
理解TCP三次握手四次挥手，理解连接池的作用和配置方法。

### 核心知识点

```
TCP三次握手 = SYN → SYN+ACK → ACK（确认双方都能通信）
TCP四次挥手 = FIN → ACK → FIN → ACK（各自确认发完收完）
连接池 = 复用连接，省去握手挥手时间
```

### 关键代码解析

**1. 查看Session的连接池配置**

```python
from requests.adapters import HTTPAdapter

adapter = HTTPAdapter(
    pool_connections=10,  # 最多10个连接池
    pool_maxsize=20       # 每个池最多20个连接
)
session.mount('http://', adapter)
session.mount('https://', adapter)
```

**2. 配置重试策略**

```python
from urllib3.util.retry import Retry

retry_strategy = Retry(
    total=3,                    # 总重试次数
    backoff_factor=1,           # 重试间隔（1s, 2s, 4s...）
    status_forcelist=[429, 500, 502, 503, 504]  # 遇到这些状态码重试
)
adapter = HTTPAdapter(max_retries=retry_strategy)
```

**3. 连接复用性能对比**

```python
# 不复用：每次新建连接
for _ in range(5):
    requests.get(url)

# 复用：保持连接
session = requests.Session()
for _ in range(5):
    session.get(url)
```

### 运行方式

```bash
python code/05_connection_pool.py
```

## 06_headers_analysis.py - HTTP头分析

### 教学目标
掌握常见请求头和响应头的作用，学会分析抓包结果。

### 核心知识点

```
请求头：标识类（Host/User-Agent）、内容协商类（Accept/Encoding）、认证类（Authorization/Cookie）
响应头：内容类（Content-Type/Length）、缓存类（Cache-Control/ETag）、安全类
```

### 关键代码解析

**1. 完整的浏览器请求头**

```python
headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36',
    'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8',
    'Accept-Language': 'zh-CN,zh;q=0.9,en;q=0.8',
    'Accept-Encoding': 'gzip, deflate, br',
    'Connection': 'keep-alive',
    'Referer': 'https://www.google.com/',
}
```

**2. 分析响应头**

```python
response = requests.get(url)

print(f"Content-Type: {response.headers.get('Content-Type')}")
print(f"Content-Length: {response.headers.get('Content-Length')}")
print(f"Set-Cookie: {response.headers.get('Set-Cookie')}")
print(f"Cache-Control: {response.headers.get('Cache-Control')}")
```

### 运行方式

```bash
python code/06_headers_analysis.py
```

## 代码学习建议

### 学习顺序

```
1. 先跑通所有代码（知道能做什么）
       ↓
2. 理解每个代码块的原理（知道为什么）
       ↓
3. 修改代码尝试不同的玩法（知道怎么改）
       ↓
4. 结合原理详解文档，形成完整知识体系
```

### 重点练习

| 优先级 | 练习内容 | 目的 |
|--------|----------|------|
| ⭐⭐⭐ | 构造完整的浏览器请求头 | 学会伪装成浏览器 |
| ⭐⭐⭐ | Session管理Cookie登录态 | 掌握登录爬取 |
| ⭐⭐⭐ | 连接池配置和性能对比 | 理解性能优化 |
| ⭐⭐ | JWT解码和分析 | 理解认证机制 |
| ⭐ | WebSocket连接 | 了解实时数据抓取 |

### 常见问题排查

| 问题 | 原因 | 解决 |
|------|------|------|
| 403 Forbidden | User-Agent被识别 | 换真实的浏览器UA |
| 401 Unauthorized | 认证失败 | 检查Token/Cookie |
| 429 Too Many Requests | 请求太频繁 | 降低并发、加间隔 |
| SSL证书错误 | 证书验证失败 | verify=False（测试） |
