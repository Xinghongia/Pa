## 常见踩坑记录

### 坑1：HTTPS证书验证失败

**问题现象**：

```
requests.exceptions.SSLError: HTTPSConnectionPool(host='xxx', port=443): Max retries exceeded
```

**原因**：

- 证书过期
- 自签名证书
- 证书链不完整

**解决方案**：

```python
# 忽略证书验证（测试用）
response = requests.get(url, verify=False)

# 指定CA证书
response = requests.get(url, verify='/path/to/ca-bundle.crt')
```

**如何避免**：

- 测试环境可以用verify=False
- 生产环境一定要用真实证书
- 定期更新CA证书包

---

### 坑2：Cookie过期导致401

**问题现象**：

```
正常爬取一段时间后，突然返回空数据或401
```

**原因**：

- Cookie有有效期（通常1小时到1天）
- 服务器端主动使Cookie失效
- 登录态被顶下去

**解决方案**：

```python
def request_with_relogin(session, url, **kwargs):
    response = session.get(url, **kwargs)

    if response.status_code == 401:
        # 重新登录
        session.post('/login', data=credentials)
        # 重试原请求
        response = session.get(url, **kwargs)

    return response
```

**如何避免**：

- 记录登录时间，快过期前主动刷新
- 捕获401状态码，自动重新登录
- 使用Refresh Token机制

### 坑3：代理突然失效

**问题现象**：

```
requests.exceptions.ProxyError: HTTPSConnectionPool... Connection refused
```

**原因**：

- 代理服务器宕机
- 代理IP被封
- 代理过期
- 代理认证失败

**解决方案**：

```python
def get_valid_proxy(proxy_manager):
    for _ in range(len(proxy_manager.proxies)):
        proxy = proxy_manager.get_next_proxy()
        try:
            # 测试代理是否可用
            response = requests.get('https://httpbin.org/ip',
                                  proxies={'http': proxy, 'https': proxy},
                                  timeout=5)
            return proxy
        except:
            proxy_manager.mark_proxy_failed(proxy)
    raise Exception("所有代理都不可用")
```

**如何避免**：

- 使用代理池，多个代理轮换
- 定期检测代理可用性
- 选择可靠的代理服务商

### 坑4：重定向导致请求失败

**问题现象**：

```
请求被重定向到其他页面，返回的内容不是预期的
```

**原因**：

- 网站会自动重定向（301/302）
- 重定向循环
- 重定向到外站

**解决方案**：

```python
# 禁止重定向
response = requests.get(url, allow_redirects=False)

# 或者手动处理重定向
if response.status_code in [301, 302]:
    new_url = response.headers['Location']
    response = session.get(new_url)
```

**如何避免**：

- 理解网站的跳转逻辑
- 使用Session保持重定向跟踪
- 设置max_redirects限制

---

### 坑5：编码问题导致乱码

**问题现象**：

```
返回的中文是乱码：ðťśňťś
```

**原因**：

- 响应编码识别错误
- Content-Type没有指定charset
- gzip压缩后解码错误

**解决方案**：

```python
# 方法1：手动指定编码
response.encoding = 'utf-8'
print(response.text)

# 方法2：自动检测编码
import chardet
detected = chardet.detect(response.content)
response.encoding = detected['encoding']

# 方法3：直接用content
print(response.content.decode('utf-8'))
```

**如何避免**：

- 优先使用response.text（自动处理）
- 遇到乱码时检查response.encoding
- 使用chardet库自动检测编码

---

### 坑6：请求超时设置不当

**问题现象**：

```
请求卡住不动，等了很久才报错
```

**原因**：

- timeout设置为None（永不超时）
- 网络慢，服务器响应慢
- 代理响应慢

**解决方案**：

```python
# 设置合理的超时时间
response = requests.get(url, timeout=(3.05, 10))
# 3.05 = 连接超时
# 10 = 读取超时

# 或者简写
response = requests.get(url, timeout=10)
```

**如何避免**：

- 根据业务设置合理的超时
- 区分连接超时和读取超时
- 使用try-except捕获超时异常

## 进阶话题

### HTTP/2 vs HTTP/1.1

| 对比 | HTTP/1.1 | HTTP/2 |
|------|----------|--------|
| 并发 | 多个TCP连接 | 单一连接多路复用 |
| 头部压缩 | 无压缩 | HPACK压缩 |
| 服务器推送 | 不支持 | 支持 |
| 二进制 | 文本 | 二进制帧 |
| 兼容性 | 广泛支持 | 现代浏览器支持 |

**爬虫中的影响**：

```
HTTP/1.1: 需要多并发连接提升速度
HTTP/2: 单连接就能高效通信
```

### QUIC协议（HTTP/3）

```
HTTP/1.1 → HTTP/2 → HTTP/3
   ↓          ↓         ↓
  文本       二进制    QUIC
             多路复用  基于UDP
```

**特点**：

- 基于UDP，更快建立连接
- 0-RTT恢复（减少握手时间）
- 更好的丢包处理

### DNS与爬虫

```
域名 → DNS解析 → IP地址 → 建立TCP连接 → 发送HTTP请求
```

**DNS缓存**：

```python
# 清除DNS缓存
import socket
socket.getaddrinfo('example.com', 80)

# 使用DNS轮换
domains = ['a.example.com', 'b.example.com', 'c.example.com']
for domain in domains:
    try:
        response = requests.get(f'http://{domain}', timeout=5)
        break
    except:
        continue
```

## 实战技巧

### 1. 如何构造完美的请求头

```python
def get_perfect_headers(url, referer=None):
    """构造完美的浏览器请求头"""
    from urllib.parse import urlparse

    parsed = urlparse(url)

    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36',
        'Accept': 'text/html,application/xhtml+xml,application/xml;q=0.9,image/webp,*/*;q=0.8',
        'Accept-Language': 'zh-CN,zh;q=0.9,en;q=0.8',
        'Accept-Encoding': 'gzip, deflate, br',
        'Connection': 'keep-alive',
        'Upgrade-Insecure-Requests': '1',
        'Sec-Fetch-Dest': 'document',
        'Sec-Fetch-Mode': 'navigate',
        'Sec-Fetch-Site': 'none' if not referer else 'same-origin',
        'Sec-Fetch-User': '?1',
    }

    headers['Host'] = parsed.netloc

    if referer:
        headers['Referer'] = referer
    else:
        headers['Referer'] = f'{parsed.scheme}://{parsed.netloc}/'

    return headers
```

### 2. 如何分析网站用的技术

```python
def detect_website_tech(url):
    """检测网站使用的技术"""
    import requests

    response = requests.get(url, timeout=10)

    tech_stack = {
        'Server': response.headers.get('Server', 'Unknown'),
        'X-Powered-By': response.headers.get('X-Powered-By', 'Unknown'),
        'Content-Type': response.headers.get('Content-Type', 'Unknown'),
        'CMS': None,
    }

    html = response.text.lower()

    if 'wordpress' in html:
        tech_stack['CMS'] = 'WordPress'
    elif 'drupal' in html:
        tech_stack['CMS'] = 'Drupal'
    elif 'ghost' in html:
        tech_stack['CMS'] = 'Ghost'

    if 'react' in html or '_next' in html:
        tech_stack['Frontend'] = 'React/Next.js'
    elif 'vue' in html:
        tech_stack['Frontend'] = 'Vue.js'
    elif 'angular' in html:
        tech_stack['Frontend'] = 'Angular'

    return tech_stack
```

### 3. 如何模拟登录获取Token

```python
def login_and_get_token(login_url, username, password):
    """模拟登录获取Token"""
    session = requests.Session()

    # 1. 获取登录页面（如果有CSRF token）
    login_page = session.get(login_url)
    csrf_token = None

    # 从页面中提取CSRF token（根据实际情况）
    import re
    match = re.search(r'name="csrf" value="([^"]+)"', login_page.text)
    if match:
        csrf_token = match.group(1)

    # 2. 发送登录请求
    login_data = {
        'username': username,
        'password': password,
    }
    if csrf_token:
        login_data['csrf'] = csrf_token

    response = session.post(login_url, data=login_data)

    # 3. 从Cookie或响应中获取Token
    token = session.cookies.get('token') or response.json().get('token')

    return session, token
```
