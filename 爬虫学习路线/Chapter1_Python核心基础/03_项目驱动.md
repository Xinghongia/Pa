# 项目A：并发新闻爬虫

## 需求说明

抓取**IT之家**新闻列表页（https://www.ithome.com/），实现以下功能：

1. **并发请求**：使用 `aiohttp` 异步抓取至少10个分类的新闻列表
2. **连接池配置**：正确配置连接池参数
3. **数据提取**：解析每条新闻的标题、链接、发布时间
4. **结果保存**：保存为 JSON 格式文件

## 技术要求

- 使用 `async/await` 实现并发
- 使用 `aiohttp.ClientSession` 管理连接
- 配置 `TCPConnector` 连接池
- 使用上下文管理器管理资源
- 正确处理异常和超时

## 验收标准

```
✅ 能够并发请求多个页面
✅ 连接池正确配置（limit, limit_per_host）
✅ 数据解析正确（标题、链接、时间）
✅ 使用 with 语句管理 Session
✅ 有异常处理，不会因为单个请求失败导致整个爬虫崩溃
✅ 结果保存为 JSON 文件
✅ 能够处理至少 100 个请求
```

# 项目B：百万级数据处理流水线

## 需求说明

实现一个**数据处理流水线**，模拟爬取100万个商品数据的场景：

1. **生成器模式**：使用生成器按需生成数据，不一次性加载到内存
2. **多阶段处理**：URL生成 → 数据爬取 → 数据清洗 → 数据统计
3. **内存监控**：监控内存占用，确保内存不爆
4. **性能统计**：统计各阶段处理速度

## 技术要求

- 使用 `yield` 实现生成器
- 使用 `yield from` 实现生成器委托
- 每个阶段都是独立的生成器
- 记录内存占用（使用 `psutil` 库）
- 统计处理速度（items/second）

## 验收标准

```
✅ 使用生成器模式处理数据
✅ 流水线各阶段独立（解耦合）
✅ 处理100万个"数据"内存不爆
✅ 实时显示处理进度
✅ 统计各阶段处理速度
✅ 可随时中断和恢复
```
