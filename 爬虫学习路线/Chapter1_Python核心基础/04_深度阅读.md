## 踩坑记录

### 3.1 异步编程常见坑

#### 坑1：忘记 `await` 导致协程对象

```python
# ❌ 错误：忘记await
async def bad_example():
    session = aiohttp.ClientSession()
    response = session.get("https://example.com")  # 返回协程对象，没有执行！
    data = await response.json()  # 这里才报错

# ✅ 正确：使用await
async def good_example():
    async with aiohttp.ClientSession() as session:
        response = await session.get("https://example.com")  # 执行请求
        data = await response.json()
```

**表现**：`await` 一个协程对象而不是 HTTP 响应。

---

#### 坑2：在同步函数中调用异步函数

```python
# ❌ 错误：在同步函数中调用async函数
def sync_caller():
    result = asyncio.run(async_func())  # 可以，但不要在async函数中嵌套
    return result

# ❌ 错误：在同步函数中直接await
def sync_caller2():
    result = await async_func()  # 语法错误！
    return result

# ✅ 正确：始终在async上下文中调用
async def async_caller():
    result = await async_func()
    return result
```

**表现**：`SyntaxError: 'await' outside async function`

---

#### 坑3：Session没有关闭导致连接泄漏

```python
# ❌ 错误：Session未关闭
async def bad_example():
    session = aiohttp.ClientSession()
    # ... 使用session
    # 如果这里抛异常，session永远不会关闭！

# ✅ 正确：使用with或ensure_future
async def good_example1():
    async with aiohttp.ClientSession() as session:
        # ... 使用session
        pass  # with会自动关闭

# ✅ 正确：后台关闭
async def good_example2():
    session = aiohttp.ClientSession()
    asyncio.create_task(session.close())  # 稍后关闭
    # ... 使用session
```

**表现**：连接数不断增加，最终达到系统限制，程序卡死。

---

#### 坑4：信号量泄漏

```python
# ❌ 错误：信号量没有释放
async def bad_example(semaphore):
    async with semaphore:
        if some_condition:
            return  # 提前return，信号量未释放！
        await do_something()

# ✅ 正确：确保信号量始终释放
async def good_example(semaphore):
    async with semaphore:
        try:
            if some_condition:
                return
            await do_something()
        finally:
            pass  # with会保证释放
```

**表现**：信号量计数减少，可用信号量耗尽，程序卡死。

---

#### 坑5：循环引用导致内存泄漏

```python
# ❌ 错误：循环引用
class Crawler:
    def __init__(self):
        self.callback = self.on_complete  # self引用

    async def on_complete(self, result):
        await self.process(result)  # 回调引用self
```

```python
# ✅ 正确：使用弱引用或避免循环
import weakref

class BetterCrawler:
    def __init__(self):
        self.callback = None

    def set_callback(self, callback):
        self.callback = weakref.WeakMethod(callback)
```

### 3.2 连接池常见坑

#### 坑6：`pool_maxsize` 太小导致阻塞

```python
# ❌ 错误：池太小
adapter = HTTPAdapter(pool_maxsize=2)
session.mount("https://api.example.com", adapter)

# 并发10个请求，只有2个能同时执行，其他排队等待
for url in urls:  # 10个URL
    session.get(url)  # 后面8个会等待

# ✅ 正确：根据并发需求设置
adapter = HTTPAdapter(pool_maxsize=50)
```

**表现**：请求变慢，看起来像"卡住"。

---

#### 坑7：`max_retries` 不生效

```python
# ❌ 错误：状态码不在重试列表中
from requests.adapters import HTTPAdapter
from urllib3.util.retry import Retry

retry = Retry(
    total=3,
    status_forcelist=[500, 502, 503, 504],  # 默认只有这些
)
adapter = HTTPAdapter(max_retries=retry)

# 429 Too Many Requests 不会重试！

# ✅ 正确：添加需要重试的状态码
retry = Retry(
    total=3,
    status_forcelist=[429, 500, 502, 503, 504],
)
```

### 3.3 生成器常见坑

#### 坑8：生成器没有迭代完就丢弃

```python
# ❌ 错误：创建生成器但不迭代
def bad_example():
    urls = generate_urls(1000000)  # 生成了，但不用
    return "done"

# ✅ 正确：使用生成器
def good_example():
    for url in generate_urls(1000000):
        if some_condition:
            break
        yield url
```

**表现**：生成器对象占用内存持续增长。

---

#### 坑9：`yield` 和 `return` 混用导致混乱

```python
def confusing_generator():
    for i in range(10):
        if i == 5:
            return i  # 相当于 raise StopIteration(i)
        yield i

# 迭代
for x in confusing_generator():
    print(x)  # 0, 1, 2, 3, 4
    # 5 时 return，StopIteration，循环结束
```

```python
# ✅ 正确：如果要返回值，用send
def generator_with_result():
    result = None
    for i in range(10):
        result = yield i
        if result is not None:
            print(f"接收到: {result}")

gen = generator_with_result()
for x in gen:
    print(x)
    if x == 3:
        gen.send("stop")  # 发送信号
```

## 常见问题解答

### Q1: 异步爬虫比同步快多少？

**A**: 理论上，异步爬虫的效率提升取决于：
- 网络延迟（延迟越高，提升越明显）
- 并发数（并发数越高，提升越高）

```
场景：100个URL，每个延迟100ms

同步：100 × 100ms = 10秒
异步（10并发）：10 × 100ms = 1秒（提升10倍）
异步（100并发）：100ms ≈ 0.1秒（提升100倍）
```

但实际中要考虑：
1. 目标服务器的限流
2. 网络带宽
3. CPU计算能力

---

### Q2: 什么时候用同步requests，什么时候用异步aiohttp？

**A**:

| 场景 | 工具 |
|------|------|
| 单次请求，脚本简单 | `requests` |
| 少量请求（<10），逻辑简单 | `requests` |
| 大量请求（>50），需要并发 | `aiohttp` |
| 需要复杂调度（优先级、超时） | `aiohttp` |
| 爬虫框架（Scrapy等） | 内置异步 |

---

### Q3: 连接池设置多大合适？

**A**: 这是一个平衡问题：

| pool_maxsize | 优点 | 缺点 |
|-------------|------|------|
| 太小（<10） | 内存占用低 | 请求排队，速度慢 |
| 适中（20-50） | 平衡 | - |
| 太大（>100） | 速度快 | 内存占用高，可能被服务器封 |

**建议**：
- 普通爬虫：10-20
- 高并发爬虫：50-100
- 访问受限网站：1-5

---

### Q4: 生成器 vs 列表，到底用哪个？

**A**:

| 数据量 | 工具 | 原因 |
|--------|------|------|
| <1000 | 列表 | 简单直接 |
| 1000-100000 | 生成器 | 内存优化 |
| >100000 | 生成器 | 必须用生成器 |

---

### Q5: 重试次数设多少合适？

**A**:

| 情况 | 重试次数 |
|------|----------|
| 稳定API | 2-3 |
| 不稳定/国际网络 | 3-5 |
| 支付等关键操作 | 5-10 |

**注意**：重试间隔要用指数退避，避免雪崩。

## 实战技巧

### 技巧1：批量创建协程

```python
import asyncio

async def task(n):
    await asyncio.sleep(0.1)
    return n

# ❌ 慢：逐个创建
tasks = []
for i in range(100):
    tasks.append(task(i))

# ✅ 快：列表推导式
tasks = [task(i) for i in range(100)]

# ✅✅ 最快：使用协程函数直接
async def run_all():
    return await asyncio.gather(*[task(i) for i in range(100)])
```

---

### 技巧2：带超时的并发

```python
import asyncio

async def with_timeout(coro, timeout):
    """为协程添加超时"""
    try:
        return await asyncio.wait_for(coro, timeout=timeout)
    except asyncio.TimeoutError:
        return None

async def main():
    tasks = [with_timeout(task(i), timeout=5) for i in range(100)]
    results = await asyncio.gather(*tasks)
```

### 技巧3：进度显示

```python
import asyncio

async def crawl_with_progress(urls):
    total = len(urls)
    completed = 0

    async def wrapped_task(url):
        nonlocal completed
        result = await fetch(url)
        completed += 1
        print(f"\r进度: {completed}/{total} ({completed*100//total}%)", end="")
        return result

    tasks = [wrapped_task(url) for url in urls]
    return await asyncio.gather(*tasks)
```
