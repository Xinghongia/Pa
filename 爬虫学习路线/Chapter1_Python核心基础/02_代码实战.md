## 代码教学指南

### 1. `01_async_crawler.py` - 异步爬虫入门

**学习顺序**：
1. 先看 `main()` 函数，理解整体流程
2. 再看 `fetch()` 协程函数，理解 `async def` 和 `await`
3. 最后理解 `asyncio.gather()` 的并发原理

**核心代码片段**：

```python
# 协程函数的定义（用async def）
async def fetch(session, url):
    async with session.get(url) as response:
        await response.read()  # await让出CPU
        return url, response.status

# 并发执行多个协程
results = await asyncio.gather(*tasks)
```

**运行效果**：
```
✅ 完成 5 个请求
⏱️  总耗时: 1.08 秒   ← 同步需要5秒，异步只用1秒！
```

### 2. `02_session_pool.py` - 连接池详解

**学习顺序**：
1. 先理解 `HTTPAdapter` 的作用
2. 再理解 `pool_maxsize` 和 `pool_connections`
3. 最后看重试策略 `Retry`

**核心代码片段**：

```python
# 配置连接池
adapter = HTTPAdapter(
    pool_connections=10,   # 维护10个不同域名的池
    pool_maxsize=20,       # 每个池最多20个连接
    max_retries=retry_strategy
)

# 配置重试策略（指数退避）
retry_strategy = Retry(
    backoff_factor=0.5,  # 失败后等 0.5s, 1s, 2s...
)
```

**运行效果**：
```

请求2: 耗时=0.089s  （复用连接，很快）
请求3: 耗时=0.075s  （复用连接，很快）
...
```

### 3. `03_context_manager.py` - 上下文管理器

**学习顺序**：
1. 先理解类的 `__enter__` 和 `__exit__`
2. 再看装饰器 `@contextmanager` 的简洁写法
3. 最后看爬虫中 Session 的正确用法

**核心代码片段**：

```python
# 类实现
class Timer:
    def __enter__(self):
        self.start = time.time()
        return self

    def __exit__(self, exc_type, exc_val, exc_tb):
        print(f"耗时: {time.time() - self.start:.2f}s")
        return False  # 不吞异常

# 装饰器实现（更简洁）
@contextmanager
def timer():
    start = time.time()
    yield  # 相当于 __enter__ 返回值
    print(f"耗时: {time.time() - start:.2f}s")
```

**爬虫中的正确用法**：

```python
# ✅ 正确：用 with 管理 Session
async with aiohttp.ClientSession() as session:
    async with session.get(url) as response:
        data = await response.json()
# with 块结束后自动关闭，不会泄漏

# ❌ 错误：Session 不关闭，连接泄漏
session = aiohttp.ClientSession()
# 如果抛异常，session 永远不会关闭！
```

### 4. `04_generator.py` - 生成器模式

**学习顺序**：
1. 先理解 `yield` 的暂停-恢复机制
2. 再对比列表和生成器的内存占用
3. 最后看爬虫中的实际应用

**核心代码片段**：

```python
# 生成器函数（用 yield）
def number_generator(n):
    for i in range(1, n + 1):
        yield i  # 暂停，返回i，然后恢复

# 使用
gen = number_generator(1000000)
print(next(gen))  # 1
print(next(gen))  # 2
# ... 每次只占几百字节内存！

# 对比列表：list(range(1000000)) 需要 8MB 内存
```

**爬虫中的实际应用**：

```python
def crawl_generator():
    """边爬边处理，不积压数据"""
    for url in generate_urls(1000000):
        data = fetch(url)
        for item in data:
            yield item  # 每爬到一条就yield出去
            # 处理完立即释放，不会内存爆炸
```

### 5. `05_encryption.py` - 加密实现

**学习顺序**：
1. 先理解哈希（MD5/SHA）的不可逆特性
2. 再看 HMAC 消息认证
3. 最后看 AES 和 RSA 的实际应用

**核心代码片段**：

```python
# 哈希（不可逆）
md5 = hashlib.md5(data.encode()).hexdigest()

# HMAC（消息认证）
signature = hmac.new(key, message.encode(), hashlib.sha256).hexdigest()

# API签名（常见模式）
def generate_sign(params, secret):
    query_string = "&".join([f"{k}={v}" for k, v in sorted(params.items())])
    sign = hmac.new(secret.encode(), query_string.encode(), hashlib.sha256).hexdigest()
    return sign
```

**对爬虫的意义**：
很多网站的加密参数（如 sign、token）都是类似逻辑
学会这些后，Chapter 6 的 JS 逆向会更简单

### 6. `06_complete_framework.py` - 综合框架

**学习顺序**：
1. 先看整体架构（`CrawlerConfig` + `AsyncCrawler`）
2. 再看各方法的分工
3. 最后理解它们如何配合

**架构设计**：

```
CrawlerConfig          # 配置类
    ├── max_concurrent    # 最大并发数
    ├── max_retries      # 最大重试次数
    └── timeout          # 超时时间

AsyncCrawler           # 爬虫类
    ├── session_context  # Session上下文管理器
    ├── fetch_with_retry # 带重试的请求（指数退避）
    ├── fetch_one        # 单个请求（受信号量控制）
    └── crawl           # 并发爬取（生成器返回结果）
```

**核心特性**：
- 信号量 `Semaphore` 控制并发数
- 指数退避策略避免请求风暴
- 生成器模式边爬边返回，内存友好
- 完善的错误处理和日志
